# AI Implementation Template

Используй этот чеклист перед началом новой задачи, чтобы ИИ-агент писал чистый и рабочий код.

## 1. Контекст
- **Фича / задача:** кратко сформулируй цель работы.
- **Исходные документы:** перечисли спецификации (`specs/<feature>/*`), задачи из Task Master и другие важные ссылки.
- **Ограничения:** версия Node, совместимость, требования по инструментам.

## 2. Планирование и дизайн
- Укажи ключевые изменения в архитектуре, маршрутах, моделях данных.
- Зафиксируй интерфейсы (request/response, схемы событий, типы).
- Перечисли предположения или открытые вопросы, требующие проверки.

## 3. Критерии качества
- **Стиль:** ESLint/Prettier, соблюдать существующую структуру импортов.
- **Логи и метрики:** что логируем, какие метрики обновляем.
- **Безопасность:** валидация входных данных, обработка ошибок, маскирование секретов.
- **Тесты:** какие тесты добавить/обновить (юнит / интеграционные / e2e) и как их запускать.

## 4. Пошаговый план
1. Шаги реализации (короткие bullet’ы).
2. Шаги тестирования (локально, CI, ручная проверка).
3. Шаги документирования (какие файлы обновить).

## 5. Определение готовности (Definition of Done)
- Код проходит `npm run lint`, `npm test` и другие обязательные проверки.
- Обновлены соответствующие документы (`docs/*`, Task Master `update-subtask`).
- Обновлены метрики/логирование где нужно.
- Ветка готова к ревью: нет лишних файлов (`dist/`, `node_modules`), все изменения описаны в заметках.

## 6. Обратная связь
- Если во время работы появились решения или обходные пути, добавь их сюда.
- Зафиксируй follow-up задачи, если что-то отложено.
